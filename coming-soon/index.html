How does it work?

Kanzan lets you focus on developing valid HTML templates first, and then define areas of content to be managed.

It generates an admin console for content managers to control just those areas of dynamic content, and outputs markup as the developer intended, free from code smells and populated with the latest content.

It fits with your workflow

As web developers, we've spent a lot of time designing our workflows. Kanzan is designed to fit into those workflows rather than replace them.

You can add Kanzan to the process you already use. It's perfect for enabling content management in sites built with tools like Jekyll, Middleman, Metalsmith or Harp, and hosted on static servers.

Manageable content areas are defined using valid HTML attributes so that your templates need no extra steps to be viewable in a browser.

your-template.html
<h1 data-kanzan-name="page-header"
     data-kanzan-type="text">
  Some page heading
</h1>

<div data-kanzan-name="strapline"
     data-kanzan-type="markdown"
     data-kanzan-label="the page strapline"
     data-kanzan-help="some guidance for the editor">
  the strapline for the page
</div>

Your markup defines your CMS

Pushing your site to Kanzan generates a custom, hosted CMS. No need to add more complex hosting infrastructure.

Content is managed on Kanzan which outputs a deployable site to your existing hosting provider.

Kanzan.io - your CMS

Publish to your existing hosting

Content authors can publish changes directly from Kanzan. A new version of the site is output and pushed to your hosting server.

At launch we will support outputting the site by pushing to a git remote. Perfect for using with free Github Pages hosting or other git-enable static hosting providers
